## 핵심 결론 (한 줄)

우리 6개 항목 구성(일주30·용신20·년주15·월주10·공망10·오행15=100)은 시중 표준과 잘 부합한다.
**벗어난 곳은 딱 하나 — 계산 끝에 무조건 +60을 더하는 baseline.** 이 때문에 모든 커플 점수가 부풀려진다.
→ 감점(마이너스)을 없애고 **0점을 바닥**으로 삼으면, 상극 문제도 baseline도 한 번에 해결된다.

## 2. 가장 큰 문제 — baseline +60

- coupleScore.ts 커밋 기록 확인 결과 "처음 생성" 커밋 하나뿐.
  → 잘 되다 틀어진 게 아니라, **처음부터 +60 구조**였다.
- 의도: "점수 낮으면 고객 실망하니 최소점 깔자."
- 부작용:
  - 모든 커플이 80~90점대로 몰려 **변별력 사라짐**.
  - 좋은 커플/별로인 커플 구분 안 됨 → 오히려 신뢰도 하락.
  - 등급 이름("반전 매력 커플" 등)으로 이미 위로 장치 있는데 점수까지 부풀려 이중으로 후함.

## 3. 감점 처리 문제

- 명리상 **상극은 무조건 나쁜 게 아님** (과한 기운을 억제해 균형 잡는 기능).
  → 큰 감점은 과하다. 상극 감점은 줄이거나 0으로 둘 여지 큼.
- 충(沖)은 실제 강한 갈등 신호라 큰 감점이 일리 있음.
- 천간 비대칭: 년간·월간은 상극이면 깎지만 상생이어도 가산 없음. 일간만 상생 +5.
  → 감점 줄일 때 이 비대칭도 함께 정리하면 일관성 생김.

### 현재 일주 감점 구조 (참고)
| 관계 | 우리 코드 | 문헌 설계안 |
|---|---|---|
| 일간 천간합 | +10 | +10 (동일) |
| 일지 육합/삼합 | +20 / +15 | A등급 +20 |
| 일지 방합/상생 | +12 | B등급 +15 |

| 일지 충/형 | -20 / -12 | D등급 0점 (바닥) |
| 일간 상극 | -8 | 감점 없음 |

## 4. 오행 궁합 차이 (가장 큰 명리적 차이)

| 구분 | 우리 코드 | 문헌 설계안 |
|---|---|---|
| 배점 | 15(+보완5) | **40 (최대 비중)** |
| 방식 | 두 사람 글자 합쳐 5행 개수 셈 | 각자 용신 구해, 상대가 그 용신 채워주는지 대조 |
| 완벽 보완 | +5 | 40 |
| 일방 보완 | — | 20 |

→ 문헌 방식이 명리적으로 더 깊음. 우리는 "개수 고른가" 수준으로 단순.

## 5. 권장 방향 (나중에 산식 정리 시)

두 직관 — "상극을 0으로" + "최소 점수를 주자" — 는 **같은 해결책**이다.
감점을 없애고 0점을 바닥으로 삼으면 상극 문제도 baseline도 동시에 풀린다.

이상적 방향 (세 방식 장점 결합):
1. **0점 바닥 철학 채택 → baseline +60 제거**
2. 오행 보완 정밀화 + 비중 확대 (용신 상호 보완 기준)
3. 조후(온도) 항목 추가 검토
4. 우리 차별화인 **MBTI·직업 가중은 유지**
5. 상극 감점 0 또는 약하게, 천간 상생/상극 비대칭 정리

> 이는 궁합 엔진 재설계라 큰 작업. **작명 점수 정리와 함께 시간 두고 진행** 권장.

## 6. 참고 메모

- 시주(태어난 시)는 궁합 점수에 반영 안 함(생년월일까지만). 시 모르는 사용자 많아 생략 흔함 → 당장 문제 아님.
- 십신(정관·정재 등) 호환 미반영. 추가 시 정밀도↑.
- MBTI 점수(mbtiScore.ts)는 환산·미입력 분기 깔끔 → **손댈 곳 없음**.
- 전문가 공통 강조: "궁합 점수는 절대 기준 아닌 참고 지표. 점수보다 납득되는 분포가 중요."
