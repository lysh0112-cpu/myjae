1. 이 작업이 무엇인가
----------------------------------------------------------------
명연재가 고객에게 내보내는 모든 AI 해설의 "말투(어투)"를, 흩어진
코드 여기저기에서 꺼내어 관리자 화면 한 곳에서 통제할 수 있게 만든
작업이다.

이전에는 사주·궁합·물상도·타로·작명 등 각 화면 코드마다 "따뜻하게
써라", "겁주지 마라" 같은 말투 지시가 따로따로 박혀 있었다. 말투를
바꾸려면 개발자가 파일마다 찾아 고쳐야 했다.

이제는 관리자 화면의 "💬 어투 관리" 탭에서 말투를 고치고 저장하면,
고객이 보는 모든 AI 해설이 한꺼번에 그 말투를 따른다.

[구조 — 핵심 개념]
각 화면의 해설 = 아래를 합쳐서 AI에게 전달한다.
  A. 공통 말투         : 모든 화면이 공유. 말투 규칙 + 쉬운 말 사전.
                         (여기서 전체 톤이 통일됨)
  B. 화면별 전용 지시문 : 그 화면에만 적용. (물상도·타로·작명·오늘운세 등)
  C. 기능 뼈대         : 코드에 남김. 계산 결과·출력 형식·안전 규칙 등
                         깨지면 안 되는 부분.

* tone_settings 테이블이 비었거나 오류가 나도, 코드에 심어둔 기본값으로
  자동 폴백되어 절대 멈추지 않는다.

5. Supabase tone_settings 테이블 구성
----------------------------------------------------------------
어투를 저장하는 단일 행(id=1) 테이블. 관리자가 어투 관리 화면에서
고치면 이 칸들이 갱신된다.

  칸 이름         | 용도
  ----------------|--------------------------------------------------
  tone_rules      | 공통 말투 규칙 (모든 해설). 6번에 "상담사 우선" 포함
  easy_terms      | 공통 쉬운 말 사전 (모든 해설)
  mulsang_guide   | 물상도(그림) 전용 지시문
  tarot_guide     | 타로(카드) 전용 지시문
  naming_guide    | 작명·이름풀이 전용 지시문
  fortune_guide   | 오늘의 운세 전용 지시문
  updated_at      | 마지막 수정 시각

Supabase SQL 편집기:
  https://supabase.com/dashboard/project/auvuwytatfcoawdrgunl/sql/new

6. 다음에 하면 좋을 것 (급하지 않음)
----------------------------------------------------------------
  1. 택일 상담사 연결
     - 출산택일·결혼택일 결과 화면의 "전문가에게 자세히 물어보기"
       버튼이 현재 alert("준비 중") 상태. 상담 흐름을 붙일 자리.
  2. 타로 상담사 연결 닫기
     - 연재 선생님 방침(타로는 고객 조회만).
       타로 결과 화면에서 상담사 버튼 숨기기.
  3. 사주 기본분석·궁합의 옛 함수 재연결 (선택)
     - buildToneBlock(코드값) → buildToneBlockFromDB(DB값)로.
       현재도 동작하나, 이걸 하면 관리자 편집분이 이 두 화면에도 완전 반영.

[전체 서비스 흐름 (참고)]
  1차 (AI 자동): 고객이 인적사항 입력 → 사주풀이·궁합·택일·개명 등
    조회 → 우리가 정한 어투로 AI 해설 자동 생성. (이번에 연결 완료)
  2차 (상담사 연결): 더 궁금하면 → 상담 상품 선택·결제 → 상담일에
    통화/채팅 → 내용 자동 수록 → 버튼으로 정리 → AI 해설 + 상담사
    설명 함께 요약(summarize) → 상담사 최종 수정 → 고객 발송.

7. 작업 방식 메모 (인계용)
----------------------------------------------------------------
  - 코드 수정: GitHub 웹 편집기에서 파일 열기 → Ctrl+A → Delete →
    새 코드 붙여넣기 → Commit changes(2번). Vercel이 자동 배포.
  - DB 칸 추가: Supabase SQL 편집기에 SQL 실행. 성공 후 "Untitled
    query" 메모는 지워도 됨(테이블·칸은 지우면 안 됨).
  - 새 화면에 어투 붙이는 표준 4단계:
      (1) SQL로 전용 칸 추가
      (2) tone API(app/api/admin/tone)에 칸 추가
      (3) ToneManager에 입력칸 추가
      (4) 해당 화면/API가 그 칸을 읽어 프롬프트에 반영
  - 화면(브라우저) 코드는 서버 함수를 직접 못 쓰므로,
    /api/admin/tone 을 fetch해서 공통 어투를 받아옴 (택일 화면 방식).
  - 파일은 하나씩 올리고 배포 확인 후 다음으로. 화면이 안 바뀌면 Ctrl+Shift+R.
