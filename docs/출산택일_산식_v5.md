## 1. 핵심 원칙

1. **부품은 재사용, 계산 로직은 안 건드린다.** 오행·신강약·통근·조후·격국은 공용 엔진(`simsanOhaeng.ts`·`yongsinNew.ts`) 재사용. 출산택일 폴더에서 조립만.
2. **모든 수치는 config.** `birthScoreConfig.ts`의 숫자만 바꾸면 산식이 자동으로 바뀐다.
3. **원문 근거 없는 규칙은 안 넣는다.** 子卯酉 삭제가 그 예.
4. **애매하면 전문가 상담으로.** 종격·극단 편중은 점수 대신 상담 안내.

---

## 2. 대상과 전제

- **대상**: 출산예정일이 정해진 분 (제왕절개·유도분만 중심)
- **철학**: 100점 날 찾기가 아니라, 극단·위험을 걸러내고 무난히 좋은 후보를 순위로.
- **중심**: 태어날 아기 원국. + 대운 타이밍(인생 흐름).
- **필수**: 의료 면책 안내. 실제 수술일은 산부인과 전문의 최우선.

---

## 3. 후보 기간·필터

- 후보 = **출산예정일 ±7일 (총 15일)** 중 **평일만** (주말·공휴일 제외 — 제왕절개 실무).
- 시간 = 수술 가능 주간 시진. 오전 선택→辰巳午 / 오후→未申 / 상관없음→辰~申.
- 각 날짜에서 좋은 시간 여러 개(최대 3개)를 보여줌. 시각마다 시주가 달라 사주·별점이 다름.

---

## 4. 점수 모델 (원국 100점 + 대운 가점)

성공축 65(격국35+통근15+재관15) : 건강축 35(조후25+오행10) + 대운 타이밍(별도 가점).

| 단계 | 항목 | 배점 | 평가 내용 | 부품 |
|---|---|---|---|---|
| 1 | 조후 | 25 | 여름생→水 / 겨울생→火 보충 | calcJohu 기준 |
| 2 | 격국 성패 | 35 | 격이 완성(성격)됐나 깨졌나(파격) | gyeokgukSungpae (신규) |
| 3 | 일간 통근 | 15 | 신강약 — 일간이 뿌리내렸나 | judgeStrength |
| 4 | 재·관 동태 | 15 | 재성·관성이 살아있나 (+부모소망) | 십신 카운트 |
| 5 | 오행 중화 | 10 | 5행 고르고 결핍·과다 없나 | simsanOhaeng+grade |
| ★ | 대운 타이밍 | +15(cap) | 용신이 30~60세 대운에 오는가 | dayunTiming (신규) |

**감점(배제 아님):**
- 월지-일지 충 → 큰 감점 (config `chungWolIl:12`)
- 조후 불능(여름 水전무 / 겨울 火전무) → 큰 감점 (`johuBulneung:15`)

**종격 분기:** 한 오행이 전체의 70%↑ 편중이면 점수 안 매기고 "전문가 상담 권함".

---

## 5. 격국 성패 자동판정 (신규 — 핵심)

**연재쌤 확정: 지장간은 보지 않고, 원국에 드러난 8글자만 센다.**

격 이름·상신은 공용 `calcGyeokguk` 재사용. 성패는 십신 카운트로 자동 판정:

| 판정 | 조건 |
|---|---|
| 관살혼잡 | 정관 있음 && 편관 있음 |
| 상관견관 | 상관 있음 && 정관 있음 |
| 재다신약 | 신약 && 재성 3개↑ |
| 무관/무재/무인 | 해당 십신 0개 |

- 파격 신호 있음 → `pagyeok` (8점)
- 파격 없고 상신 유력 → `sunggyeok` (35점)
- 그 외 → `gyeokOnly` (20점)

**2차 판정(상신 유력도·형충 무력화 등)은 보류** — 1차 배포 후 연재쌤과.

---

## 6. 대운 타이밍 (신규 — 연재쌤 강조)

> "아기 원국에 부족한 오행/용신 오행이, 대운에서 중요한 시기(30~60세)에 와주는지가 중요하다."

- **기준 오행**: 용신(억부/조후) 우선. `calcYongsinNew().eokbu.yongsin`.
- **중요 시기**: 30~60세 대운.
- **판정**: 그 구간 대운의 천간·지지가 용신이면 가점. (둘 다=hitBoth 10, 하나=hitOne 5, cap 15)
- 정지된 원국(그릇)과 별개로, "필요한 기운이 인생 핵심기에 채워지는 타이밍"을 본다.
- 대운 계산은 성별 필요(순/역행) → 입력에서 성별 필수화.

## 7. 부모 소망 반영

입력의 "바라는 점"을 재·관 항목 가중에 살짝 반영 (전부 뒤집지 않고 "일부라도").

| 소망 | 보는 것 |
|---|---|
| 재물운 | 식상생재·재성 통근 |
| 공부운·명예 | 관인상생·정관·인성 |
| 건강 | 조후 해결 |
| 인덕 | 인성·통근 |

※ v5 현재는 아직 실제 가중 미반영(다음 단계). config `wish.weight:1.3` 자리만.

## 8. 결과 화면

- 15일 중 좋은 날 **3개** 🥇🥈🥉, 각 날 좋은 시간 여러 개.
- 시간 누르면 **모달**: 한줄요약 + 별점3줄 + 원국표 + 공망 + **대운표** + 3문장 해설 + 대운흐름 + 오행분포.
- **점수·등급 숫자는 감춤.** 별점·순위·해설만. (교훈 A "점수는 상처를 준다")
- 별점 3줄: **건강**(조후+오행) / **성공**(격국+통근) / **재물·명예**(재관+대운).
- 모달 → 이미지로 저장 → 카톡 전송 가능.
- "사주 자세히 보기" → 사주보기(유료, 결제 관문 예정).

## 12. 함정·교훈 (다음 세션 주의)

- **캐시 키 통일**: 대운 캐시 키는 반드시 `padStart(2,'0')`로. 저장·조회 키 불일치로 대운표가 빈 채 나오는 버그가 있었음.
- **저장 스키마 변천**: v5 저장 구조가 여러 번 바뀜(시간1개→여러개→대운추가). 옛 보관함 기록은 새 필드가 비어 나올 수 있음(에러는 없음).
- **공용 엔진 수정 금지**: yongsinNew.ts 등은 궁합·사주보기가 공유. export 추가 외엔 건드리지 말 것.
- **원문 대조 없이 규칙 추가 금지**: 子卯酉가 그 사례(NotebookLM이 페이지번호까지 대며 지어냄).
